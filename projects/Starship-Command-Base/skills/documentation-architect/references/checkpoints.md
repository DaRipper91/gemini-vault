# Documentation Quality Checkpoints

- [ ] **Accurate**: All code examples work and descriptions match current implementation.
- [ ] **Accessible**: Important concepts are explained before their first use.
- [ ] **Consistent**: Naming conventions follow the `Style Guide`.
- [ ] **Surgical**: Changes only modify the necessary sections of existing manuals.
- [ ] **Hyperlinked**: All internal references to other documentation files are correct.
- [ ] **Technical**: The persona reflects a Senior Software Engineer.
- [ ] **Complete**: Both User and Developer guides have been updated if the feature touches both UI and Logic.
